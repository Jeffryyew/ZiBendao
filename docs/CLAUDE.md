# CLAUDE.md — 资本道 Zibendao 项目工作约定

> 本文件供 Claude Code 每次会话开始时读取。动手前请先读完，并遵守以下约定。

## 项目身份

- **产品**：资本道 Zibendao — 企业资本操作系统（Capital Operating System）
- **使命**：Transform Businesses Into Investable Enterprises（助力企业转型，成为值得投资的对象）
- **运营公司**：Eutopos Equity Sdn Bhd
- **部署**：Vercel（git 提交后自动部署）

**它不是** ERP、CRM 或会计软件。任何改动都不得让产品滑向这类风格。

## 核心用户旅程（产品脊柱）

```
财务规范化 → 企业资本化 → 融资化 → 投资化
```

## 设计哲学与风格

- 复杂逻辑隐藏在后台，**用户永远只看到下一步**。
- 视觉风格：Apple（简洁高级）+ Notion（结构清晰）+ Duolingo（成长式正反馈）。

## 三大核心模块（现有架构）

1. **学习系统** — 课程系统基础架构已搭建。
2. **企业数据** — 模块、单一公司模式、集团模式已就位。
3. **资本工具** — 基础架构已搭建。

已上线能力：用户注册 / 登录 / Google Login、Dashboard、企业数据（单一+集团）、企业成长状态、成就徽章系统。

## 7 条开发原则（每次开发前必读）

1. 不破坏现有功能。
2. 保持极简设计。
3. 优先提升用户体验。
4. 避免 ERP 与会计软件风格。
5. 所有功能围绕企业资本成长设计。
6. 用户永远只看到下一步。
7. 优先优化现有功能，而不是增加复杂功能。

## 工作流（每次收到需求）

1. 先分析现有架构。
2. 评估影响范围。
3. 提出最佳方案。
4. 经确认后再修改。

**完成后必须说明**：修改内容 · 测试结果 · 风险说明 · 下一步建议。
除非明确要求，**不要推翻现有设计或重构整个系统**。

## 看板约定（最重要）

本项目以 `docs/资本道网站开发总看板.md` 为**单一事实来源（Single Source of Truth）**。

- **动代码前**：先读《总看板》，确认当前阶段、闸门与卡片状态。
- **改动后**：更新《总看板》——勾选/移动对应卡片，并在「决策日志」追加一行（日期、变更、影响范围），更新顶部「最近更新」日期。
- 新功能默认进《总看板》的**想法池**，经影响评估后才进排期，避免功能膨胀。

## 红线与闸门

- **AI 红线**：能用系统/规则/公式解决的，不要用 AI；AI 仅作顾问（解释/分析/建议/引导）。AI 功能前必须有「使用同意层」(默认关闭、提醒数据将发给第三方)。详见 AI 原则。
- **合规红线**：凡涉及投资/财务建议的输出（尤其 Investability Score、AI 财务建议），必须先有免责声明才能对真实用户开放。
- **闸门 A**：课程内容未定稿（P2），不进入学习体验/AI 课程导师等后续阶段。
- **闸门 B**：邮箱验证码 / OTP / 支付 / 邮件系统等，待公开正式域名上线后再做。
